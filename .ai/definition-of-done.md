# Definition of Done (per phase)

PHASE: THINKING
- project.context.md exists and validated
- primary user action defined

PHASE: ARCHITECTURE
- file-tree proposed and approved
- server/client split documented
- data model draft present

PHASE: IMPLEMENTATION
- server logic implemented + unit tests
- tsc --noEmit passes
- lint passes

PHASE: UX
- loading/empty/error states present
- accessibility checklist (contrast, labels) applied

PHASE: REVIEW
- Vulnerability scan completed (no critical issues)
- Human review approved

Only after all above -> Merge & Deploy
